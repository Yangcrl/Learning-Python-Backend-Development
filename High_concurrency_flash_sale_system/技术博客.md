# 从0到1构建秒杀系统：Redis预减库存与Lua脚本

## 前言

秒杀是电商系统中常见的营销活动，也是对系统高并发处理能力的一种考验。在秒杀场景中，如何保证库存数据的一致性，防止超卖，同时保证系统的高性能和稳定性，是每个开发者都需要面对的挑战。

本文将详细介绍如何使用Redis预减库存和Lua脚本来构建一个高并发的秒杀系统，从设计到实现，从理论到实践，帮助你理解秒杀系统的核心原理。

## 一、秒杀系统的核心挑战

### 1. 高并发请求

秒杀活动通常会吸引大量用户同时参与，系统需要处理短时间内的高并发请求，这对系统的性能提出了很高的要求。

### 2. 库存一致性

在高并发场景下，如何保证库存数据的一致性，防止超卖，是秒杀系统的核心挑战之一。如果处理不当，可能会导致实际库存为负数，或者多个用户抢购到同一件商品。

### 3. 防止恶意请求

秒杀活动容易受到恶意攻击，如刷单、爬虫等，需要采取措施防止这些恶意请求对系统造成影响。

## 二、Redis预减库存方案

### 1. 什么是预减库存

预减库存是指在秒杀活动开始前，将秒杀商品的库存提前加载到Redis中，当用户请求秒杀时，直接从Redis中扣减库存，而不是每次都访问数据库。

### 2. 预减库存的优势

- **减少数据库压力**：Redis的性能远高于数据库，能够处理更高的并发请求
- **提高响应速度**：Redis的响应时间比数据库快得多
- **简化库存管理**：将库存管理集中在Redis中，减少了数据库的负担

### 3. 预减库存的实现

#### 3.1 数据结构设计

使用Redis的Hash结构存储秒杀活动的库存信息：

```
# 键名格式
seckill:stock:{activity_id}

# 字段
stock: 秒杀库存
product_id: 商品ID
seckill_price: 秒杀价格
```

#### 3.2 库存预热脚本

```python
# preheat.py
import redis
from app.core.redis_client import pool
from app.core.database import get_db
from app.models import SeckillActivity

# 连接Redis
redis_client = redis.Redis(connection_pool=pool, decode_responses=True)

# 获取数据库会话
db = next(get_db())
try:
    # 查询所有进行中的秒杀活动
    activities = db.query(SeckillActivity).filter(SeckillActivity.status == 2).all()
    
    print(f"发现 {len(activities)} 个进行中的秒杀活动")
    
    for activity in activities:
        # 构建Redis键名
        stock_key = f"seckill:stock:{activity.id}"
        
        # 检查库存是否已存在
        if redis_client.exists(stock_key):
            current_stock = redis_client.hget(stock_key, "stock")
            print(f"活动 {activity.id} 的库存已存在于Redis中，当前库存: {current_stock}")
        else:
            # 设置库存信息
            redis_client.hset(stock_key, "stock", activity.seckill_stock)
            redis_client.hset(stock_key, "product_id", activity.product_id)
            redis_client.hset(stock_key, "seckill_price", activity.seckill_price)
            
            # 设置过期时间（活动结束后2小时）
            expire_time = int((activity.end_time - datetime.datetime.now()).total_seconds()) + 7200
            redis_client.expire(stock_key, expire_time)
            
            print(f"活动 {activity.id} 的库存已预热到Redis中，初始库存: {activity.seckill_stock}")
    
    print("\n库存预热完成！")
finally:
    db.close()
```

## 三、Lua脚本实现原子库存扣减

### 1. 为什么使用Lua脚本

在高并发场景下，库存扣减操作需要保证原子性，否则可能会导致超卖。Lua脚本在Redis中是原子执行的，能够确保库存扣减操作的一致性。

### 2. Lua脚本实现

```lua
-- stock.lua
local stock_key = KEYS[1]
local stock = tonumber(redis.call('hget', stock_key, 'stock'))

if stock and stock > 0 then
    redis.call('hincrby', stock_key, 'stock', -1)
    return tonumber(redis.call('hget', stock_key, 'stock'))
else
    return -1
end
```

### 3. 脚本解析

- `local stock_key = KEYS[1]`：获取传入的库存键名
- `local stock = tonumber(redis.call('hget', stock_key, 'stock'))`：获取当前库存
- `if stock and stock > 0 then`：检查库存是否充足
- `redis.call('hincrby', stock_key, 'stock', -1)`：原子扣减库存
- `return tonumber(redis.call('hget', stock_key, 'stock'))`：返回剩余库存
- `else return -1`：库存不足时返回-1

### 4. 在Python中使用Lua脚本

```python
# 加载Lua脚本
def load_lua_script():
    with open("lua/stock.lua", "r", encoding="utf-8") as f:
        return f.read()

# 执行脚本扣减库存
lua_script = load_lua_script()
script = redis_client.register_script(lua_script)
result = script(keys=[stock_key])

if result == -1:
    raise HTTPException(status_code=400, detail="库存不足")
```

## 四、分布式锁防止超卖

### 1. 什么是分布式锁

分布式锁是在分布式系统中用于协调多个节点对共享资源访问的机制，确保在分布式环境下，同一时间只有一个进程能够访问共享资源。

### 2. Redis实现分布式锁

```python
# redis_lock.py
import redis
import uuid

class RedisLock:
    def __init__(self, redis_client, key, expire=10):
        self.redis_client = redis_client
        self.key = key
        self.expire = expire
        self.identifier = str(uuid.uuid4())
    
    def acquire(self):
        result = self.redis_client.set(
            self.key, 
            self.identifier, 
            nx=True, 
            ex=self.expire
        )
        return result is True
    
    def release(self):
        lua_script = """
        if redis.call('get', KEYS[1]) == ARGV[1] then
            return redis.call('del', KEYS[1])
        else
            return 0
        end
        """
        lua_script = lua_script.replace('ARGV[1]', f'"{self.identifier}"')
        result = self.redis_client.eval(lua_script, 1, self.key)
        return result == 1
```

### 3. 在秒杀接口中使用分布式锁

```python
# 构建锁键和库存键
lock_key = f"seckill:lock:{activity_id}"
stock_key = f"seckill:stock:{activity_id}"

# 获取分布式锁
lock = RedisLock(redis_client, lock_key, expire=5)
if not lock.acquire():
    raise HTTPException(status_code=429, detail="系统繁忙，请稍后重试")

try:
    # 加载并执行Lua脚本
    lua_script = load_lua_script()
    script = redis_client.register_script(lua_script)
    
    # 执行脚本扣减库存
    result = script(keys=[stock_key])
    
    if result == -1:
        raise HTTPException(status_code=400, detail="库存不足")
    
    # 生成订单号
    order_id = str(uuid.uuid4())
    
    # 返回订单号和剩余库存
    return {
        "order_id": order_id,
        "remaining_stock": result,
        "message": "秒杀成功"
    }
finally:
    # 释放锁
    lock.release()
```

## 五、完整的秒杀流程

### 1. 系统启动时

1. 执行库存预热脚本，将秒杀活动的库存加载到Redis中
2. 初始化布隆过滤器，加载商品ID，防止缓存穿透

### 2. 用户请求秒杀时

1. 检查秒杀活动是否存在且进行中
2. 获取分布式锁
3. 调用Lua脚本原子扣减库存
4. 生成订单号
5. 释放分布式锁
6. 返回秒杀结果

### 3. 秒杀活动结束后

1. Redis中的库存数据自动过期
2. 系统恢复正常状态

## 六、性能优化措施

### 1. 缓存优化

- **商品详情缓存**：将商品详情缓存到Redis中，减少数据库查询
- **布隆过滤器**：使用布隆过滤器防止缓存穿透
- **缓存预热**：在系统启动时预热缓存，减少首屏加载时间

### 2. 数据库优化

- **连接池**：使用数据库连接池，减少连接创建和销毁的开销
- **索引优化**：为查询频繁的字段创建索引
- **读写分离**：将读操作和写操作分离，提高系统性能

### 3. 接口优化

- **限流**：使用Redis实现滑动窗口限流，防止恶意请求
- **异步处理**：使用异步IO，提高并发处理能力
- **降级策略**：当系统负载过高时，启用降级策略，保证核心功能的正常运行
