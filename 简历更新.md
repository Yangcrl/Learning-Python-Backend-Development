## 项目经历

### 高并发秒杀系统

**项目描述**：
一个基于微服务架构的高并发秒杀系统，支持商品秒杀、订单管理、用户认证等功能，能够处理每秒 hundreds 级的并发请求。

**技术栈**：
- **后端**：FastAPI, Python 3.12, gRPC
- **数据库**：MySQL 8.0
- **缓存**：Redis 6.2
- **消息队列**：RabbitMQ 3.11
- **服务发现**：Consul 1.15
- **容器化**：Docker, Kubernetes
- **监控**：Prometheus, Grafana
- **压测**：Locust

**核心功能**：
- 商品管理：商品信息的CRUD操作，支持商品详情查询和库存管理
- 秒杀活动：支持秒杀活动的创建、查询和管理
- 秒杀接口：高并发下的秒杀接口，支持库存预减和异步下单
- 订单管理：订单的创建、查询和管理，支持订单超时取消
- 用户管理：用户注册、登录和认证
- 服务发现：微服务之间的服务发现和通信
- 自动扩缩容：基于CPU利用率的自动扩缩容

**技术亮点**：
1. **高并发处理**：
   - 使用Redis缓存减少数据库查询，QPS从41.7提升到128.3，提升207.9%
   - 使用RabbitMQ实现异步下单，提高系统吞吐量
   - 使用Redis Lua脚本实现原子操作，防止超卖
   - 使用分布式锁确保并发安全

2. **系统可靠性**：
   - 使用死信队列处理订单超时取消，确保库存回滚
   - 使用布隆过滤器防止缓存穿透
   - 使用数据库连接池提高数据库连接效率
   - 使用服务健康检查确保服务可用性

3. **系统可扩展性**：
   - 微服务架构，独立部署和扩展
   - Kubernetes HPA自动扩缩容，基于CPU利用率50%的目标
   - 服务发现和负载均衡
   - 容器化部署，便于水平扩展

4. **系统可监控性**：
   - Prometheus监控系统指标
   - Grafana可视化监控数据
   - 服务健康检查和告警

**项目成果**：
- 性能提升：QPS提升207.9%，平均响应时间减少67.5%
- 系统稳定性：零失败请求，订单超时自动取消，库存回滚机制

## 面试话术准备

### 项目介绍

**Q：请介绍一下你参与的高并发秒杀系统项目。**

A：我参与的高并发秒杀系统是一个基于微服务架构的电商系统，主要用于限时特价商品的抢购活动。系统采用了FastAPI作为后端框架，MySQL作为数据库，Redis作为缓存，RabbitMQ作为消息队列，Consul作为服务发现，Kubernetes作为容器编排平台。

系统的核心功能包括商品管理、秒杀活动管理、秒杀接口、订单管理、用户管理等。通过一系列的优化措施，系统性能得到了显著提升，QPS从41.7提升到128.3，平均响应时间从240ms减少到78ms。

### 技术亮点

**Q：项目中有哪些技术亮点？**

A：项目的技术亮点主要包括：

1. **高并发处理**：使用Redis缓存减少数据库查询，使用RabbitMQ实现异步下单，使用Redis Lua脚本实现原子操作防止超卖，使用分布式锁确保并发安全。

2. **系统可靠性**：使用死信队列处理订单超时取消，使用布隆过滤器防止缓存穿透，使用数据库连接池提高数据库连接效率，使用服务健康检查确保服务可用性。

3. **系统可扩展性**：采用微服务架构，独立部署和扩展；使用Kubernetes HPA实现自动扩缩容；使用服务发现和负载均衡；采用容器化部署，便于水平扩展。

4. **系统可监控性**：使用Prometheus监控系统指标，使用Grafana可视化监控数据，实现服务健康检查和告警。

### 技术挑战

**Q：在项目中遇到了哪些技术挑战？如何解决的？**

A：在项目中遇到的主要技术挑战包括：

1. **高并发下的超卖问题**：使用Redis Lua脚本实现原子操作，确保库存预减的原子性；使用分布式锁防止并发操作导致的超卖；使用消息队列实现异步下单，减少系统压力。

2. **缓存穿透问题**：使用布隆过滤器过滤无效请求；缓存空值，防止缓存穿透；实施限流措施，防止恶意请求。

3. **订单超时取消**：使用RabbitMQ死信队列处理订单超时；实现库存回滚机制；定期检查订单状态。

4. **服务间通信**：使用gRPC实现服务间通信；使用Consul实现服务发现；实现服务健康检查。

### 性能优化

**Q：项目中做了哪些性能优化？效果如何？**

A：项目中的性能优化主要包括：

1. **缓存优化**：使用Redis缓存商品详情和库存，实现缓存预热，使用布隆过滤器防止缓存穿透。

2. **数据库优化**：配置数据库连接池，优化索引，优化SQL查询语句。

3. **服务优化**：微服务拆分，服务发现，API网关，负载均衡。

4. **并发优化**：分布式锁，消息队列，异步处理。

优化效果：QPS从41.7提升到128.3，提升207.9%；平均响应时间从240ms减少到78ms，减少67.5%；秒杀接口在200并发用户下保持稳定，零失败请求。
